# Network

<details>
<summary>1. 인터넷 (Internet)</summary>

<br/>

> **개념** : TCP/IP 프로토콜 기반 / 컴퓨터, 네트워크들이 연결된 광범위한 컴퓨터 통신망

<br/>

### 1) IP 주소(Internet Protocol Address) - IPv4

> **개념** : 컴퓨터 자원 구분하기 위한 <u>고유 주소</u>
>
> - 32bit = 8bit * 4부분
> - A ~ E 클래스
>    - A : 국가, 대형 통신망 / 0 ~ 127 / 2의 24승 호스트 (3부분 사용 가능)
>    - B : 중대형 통신망 / 128 ~ 191 / 2의 16승 호스트 (2부분 사용 가능)
>    - C : 소규모 통신망 / 192 ~ 223 / 2의 8승 호스트 (1부분 사용 가능)
>    - D : 멀티캐스트용(화상 회의) / 224 ~ 239
>    - E : 실험적 주소, 공용 X

<br/>

### 2) 서브네팅(Subnetting)

> **개념** : 할당된 네트워크 주소 => 여러 개의 작은 네트워크로 나누어 사용하는 것
>
> => IP 주소 계산 문제

<br/>

> 💡 **서브넷 마스크(Subnet Mask)** : IP 주소 중 네트워크 주소 / 호스트 주소 구분하기 위한 비트
>
> - 클래스마다 다르게 사용
> - 어떤 거를 써도 되는 지 / 쓰면 안되는 지 구분하기 위해 사용
>    - 1 : 사용 X (네트워크 주소)
>    - 0 : 사용 O (호스트 주소)

<br/>

> 💡 서브넷 나눴을 때, 각 네트워크에 대해
> - 첫 번째 주소 : 네트워크 IP 주소
> - 마지막 주소 : 브로드캐스트 IP 주소

<br/>

### 3) IPv6 (Internet Protocol version 6)

> IPv4 체계의 주소 부족 문제 해결을 위해 개발 (IPv4와 호환 good)
>
> - 128bit = 16bit * 8부분 (콜론 : 으로 구분)
> - 장점
>    - IPv4 보다 빠른 전송 속도
>    - 보안 문제 해결 (인증성, 기밀성, 데이터 무결성)
>    - 실시간 흐름 제어 => 향상된 멀티미디어 기능
>    - 주소의 <u>확장성</u>, 융통성, 연동성 뛰어남
>    - 패킷 크기 <u>확장</u> 가능 => 패킷 크기에 제한 X
>    - 기본 헤더 + <u>확장</u> 헤더 => 다양한 정보 저장 가능 => 네트워크 기능 확장 용이

<br/>

> 💡 **IPv6의 3가지 주소 체계**
>
> - **유니캐스트 (Unicast)** - 1:1 통신
> - **멀티캐스트 (Multicast)** - 1:다 통신
> - **애니캐스트 (Anycast)** - 1:1 통신 (가장 가까이 있는 송/수신자 간)

<br/>

### 4) 도메인 네임(Domain Name)

> **개념** : 숫자로 된 IP 주소 => 문자 형태로 표현
>
> **도메인 네임 - 구성**
> - 호스트 컴퓨터 이름.소속 기관 이름.소속 기관 종류.소속 국가명 (오른쪽이 상위 도메인)
> - ex) www.sinagong.co.kr

<br/>

> 💡 **DNS (Domain Name System) / DNS 서버**
> 
> - **DNS (Domain Name System)** : 문자로 된 도메인 네임 => IP 주소로 변환하는 시스템
> - **DNS 서버** : 위 역할하는 서버

<br/>

<summary>2. OSI (Open System Interconnection) 참조 모델</summary>

<br/>

> **개념** : (시스템 간 원활한 통신을 위해) ISO(국제표준화기구)에서 제안한 통신 규약(Protocol) / 컴퓨터 통신망의 논리 구조
>
> **OSI 7계층** (물데네전세표응)
> - 하위 계층 : 물리 - 데이터 링크 - 네트워크
> - 상위 계층 : 전송 - 세션 - 표현 - 응용

<br/>

### 1) 물리 계층 (Physical Layer)

> **개념** : <u>장치 간 접속/절단 등</u> 기계적/기능적/절차적 특성에 대한 규칙 정의
>
> - 물리적 전송 매체, 전송 신호 방식 정의
> - 표준 : RS-232C, X.21 등
> - 장비 : 리피터, 허브

<br/>

### 2) 데이터 링크 계층 (Data Link Layer)

> **개념** : <u>인접한 개방 시스템 간 연결 설정/유지/종료</u>
>
> - <u>흐름 제어 / 오류 제어</u>
> - 프레임 동기화 (프레임 시작, 끝 구분) / 순서 제어(프레임 순서적 전송)
> - 표준 : HDLC, LAPB, LLC, MAC, LAPD, PPP 등
> - 장비 : 랜카드, 브리지, 스위치

<br/>

### 3) 네트워크 계층 (Network Layer, 망 계층)

> **개념** : <u>개방 시스템 간 네트워크 연결 관리</u>
>
> - 네트워크 연결 설정/유지/해제
> - <u>경로 설정(Routing) / 트래픽 제어 / 패킷 정보 전송</u> / 데이터 교환, 중계
> - 표준 : X.25, IP 등
> - 장비 : 라우터

<br/>

### 4) 전송 계층 (Transport Layer)

> **개념** : <u>종단 시스템(End-to-End = 단말기) 간</u> 투명한 데이터 전송
>
> - OSI 하위 3계층 - 상위 3계층의 인터페이스(Interface) 담당
> - 주소 설정 / 종단 시스템 간 연결 설정, 해제 / 균일한 데이터 전송
> - <u>흐름 제어 / 오류 제어</u> / 다중화(분할, 재조립)
> - 표준 : TCP, UDP 등
> - 장비 : 게이트웨이

<br/>

### 5) 세션 계층 (Session Layer)

> **개념** : 송/수신측 간 관련성 유지
>
> - 대화 구성 / 동기 제어 / 데이터 교환 관리
> - <u>토큰 사용</u> (대화 생성/관리/종료 위해)

> ✅ **동기점** : 오류 데이터의 회복을 위해 사용
> - 소동기점
> - 대동기점

<br/>

### 6) 표현 계층 (Presentation Layer)

> **개념** : 데이터 형태 변환 (응용 - 세션 계층 간)
>
> - <u>코드 변환 / 정보 형식 변환</u>
> - 데이터 암호화/압축
> - 구문 검색 / 문맥 관리

<br/>

### 7) 응용 계층 (Application Layer)

> **개념** : 사용자(응용 프로그램)가 OSI 환경에 접근할 수 있게 서비스 제공
>
> - 응용 프로세스 간 정보 교환
> - 전자 사서함 / 파일 전송 / 가상 터미널 등

<br/>

</details>

<summary>3. TCP / IP</summary>

<br/>

### 1) 프로토콜 (Protocol)

> **개념** : 서로 다른 기기 간, 원활한 데이터 교환을 위해, 표준화한 통신 규약
>
> **프로토콜 - 기본 요소**
>
> - **구문(Syntax)** : 전송하려는 데이터 형식, 부호화, 신호 레벨 등 규정
> - **의미(Semantics)** : (기기 간 효율, 정확한 정보 전송을 위한) 협조 사항 / (오류 관리 위한) 제어 정보 규정
> - **시간(Timing)** : 기기 간 통신 속도, 메시지 순서 제어 규정
>
> **프로토콜 - 기능**
>
> - 단편화, 재결합
> - 캡슐화
> - 다중화
> - 흐름 제어, 오류 제어, 순서 제어
> - 주소 지정, 경로 제어
> - 동기화, 전송 서비스

<br/>

### 2) TCP / IP (Transmission Control Protocol / Internet Protocol)

> **개념** : 서로 다른 기종의 컴퓨터가 데이터 주고 받을 수 있게 하는 <u>표준 프로토콜</u>
>
> **TCP**
> - OSI 7계층 중 '전송' 계층
> - 가상 회선 방식 기반 / 연결형 서비스
> - 패킷 다중화
> - 순서 제어, 오류 제어, 흐름 제어
>
> **IP**
> - OSI 7계층 중 '네트워크' 계층
> - 데이터그램 방식 기반 / 비연결형 서비스
> - 패킷 분해 / 조립
> - 주소 지정, 경로 선택 (Routing)

<br/>

### 3) TCP / IP 구조

> **1. 응용 계층** (OSI : 응용+표현+세션)
> - 응용 프로그램 간 데이터 송/수신
> - TELNET, FTP, SMTP, SNMP, DNS, HTTP
>
> **2. 전송 계층** (OSI : 전송)
> - 호스트 간 신뢰성 있는 통신
> - TCP, UDP, RTCP
>
> **3. 인터넷 계층** (OSI : 네트워크)
> - 주소 지정, 경로 설정
> - IP, ICMP, IGMP, ARP, RARP
>
> **4. 네트워크 액세스 계층** (OSI : 데이터 링크+물리)
> - Ethernet, IEEE 802, HDLC, X.25, RS-232C, ARQ

<br/>

### 4) 응용 계층 - 주요 프로토콜

> **FTP(File Transfer Protocol)** : 컴퓨터 - 컴퓨터/인터넷 간 원격 파일 전송 프로토콜
>
> **SMTP(Simple Mail Transfer Protocol)** : 전자 우편 교환
>
> **TELNET** : 원격 접속 / 프로그램 실행 등 시스템 관리 작업 할 수 있는 가상의 터미널(Virtual Terminal) 기능
>
> **SNMP(Simple Network Management Protocol)** : 네트워크 관리 프로토콜 / 네트워크 기기(라우터, 허브)의 네트워크 정보를 네트워크 관리 시스템에 전송할 때의 표준 통신 규약
>
> **DNS(Domain Name System)** : 도메인 네임 - IP 주소 매핑(Mapping)
>
> **HTTP(HyperText Transfer Protocol)** : WWW에서 HTML 문서 송/수신하기 위한 표준 프로토콜

<br/>

### 5) 전송 계층 - 주요 프로토콜

> **TCP(Transmission Control Protocol)**
> - 양방향 연결형 서비스 / 가상 회선 연결 형태
> - 스트림 위주의 전달 (패킷 단위)
> - 신뢰성 있는 경로 확립 / 투명성이 보장되는 통신
> - 순서 제어, 오류 제어, 흐름 제어
> - 헤더 = 20 ~ 60Byte (최대 100Byte)
>
> **UDP(User Datagram Protocol)**
> - 비연결형 서비스
> - TCP보다 단순한 헤더 => 오버헤드 적고, 전송 속도 빠름
> - 흐름 제어, 순서 제어 X
> - 빠른 속도 필요 / 여러 사용자에게 데이터 동시 전달 / 정기적 반복 전송 시 사용
> - 실시간 전송에 유리 / 신뢰성 < 속도
> - 헤더 = Source Port Number, Destination Port Number, Length, Checksum 등 포함
>
> **RTCP(Real-Time Control Protocol)**
> - RTP(Real-time Transport Protocol) 패킷의 전송 품질 제어 프로토콜
> - 세션 참여자들에게 주기적으로 '제어 정보' 전송
> - 하위 프로토콜은 데이터 패킷, 제어 패킷의 다중화 제공
> - 데이터 전송 모니터링 / 최소한의 제어, 인증 기능
> - RTCP 패킷 : 32비트의 경계

<br/>

### 6) 인터넷 계층 - 주요 프로토콜

> **IP(Internet Protocol)** : 주소 지정, 경로 설정 / 비연결형인 데이터그램 방식, 신뢰성 보장 X
>
> **ICMP(Internet Control Message Protocol, 인터넷 제어 메시지 프로토콜)** : IP와 조합해 (통신 중 오류 처리, 전송 경로 변경 등 위한) 제어 메시지 관리 / 헤더는 8Byte
>
> **IGMP(Internet Group Management Protocol, 인터넷 그룹 관리 프로토콜)** : 호스트, 라우터 사이에서 멀티 캐스트 그룹 유지
>
> **ARP(Address Resolution Protocol)** : 호스트 IP 주소 -> 물리적 주소(MAC Address)
>
> **RARP(Reverse Address Resolution Protocol)** : 물리적 주소 -> IP 주소

<br/>

### 7) 네트워크 액세스 계층 - 주요 프로토콜

> **Ethernet (IEEE 802.3)** : CSMA/CD 방식의 LAN
>
> **IEEE 802** : LAN을 위한 표준 프로토콜
>
> **HDLC** : 비트 위주의 데이터 링크 제어 프로토콜
>
> **X.25** : 패킷 교환망을 통한 DTE - DCE 간 인터페이스 제공
>
> **RS-232C** : 공중 전화 교환망(PSTN) 통한 DTE - DCE 간 인터페이스 제공
>
> **L2TP(Layer 2 Tunneling Protocol)** : L2F + PPTP 기술적 장점 결합한 터널링 프로토콜 / 자체 암호화, 인증 기능 없어서 다른 보안 프로토콜과 함께 사용

<br/>

</details>



<details>
<summary>4. 네트워크 구축</summary>

<br/>

### 1) 네트워크(Network)

> **개념** : 두 대 이상의 컴퓨터를 연결하여 자원을 공유하는 것
>
> **목적** : (다른 컴퓨터의) 데이터, 프로그램, 주변장치, 인터넷 등 공유하기 위해

<br/>

> **네트워크 설치 구조(구성 형태 / 토폴로지, Topology)** : (물리적 위치에 따라) 성형, 링형, 버스형, 계층형, 망형
>
> **네트워크 분류** : (사이트가 분포된 지리적 범위에 따라) LAN, WAN

<br/>

### 2) 성형(Star, 중앙 집중형)

> **개념** : 중앙 컴퓨터 중심으로 단말장치 연결 (중앙집중식 네트워크 구성)
>
> - Point-to-Point 방식 회선 연결
> - 단말장치 추가/제거 easy
> - 중앙 컴퓨터 고장나면 전체 통신망 정지

<br/>

### 3) 링형(Ring, 루프형)

> **개념** : Point-to-Point 방식 / 이웃하는 컴퓨터, 단말장치 간 연결
>
> - 단말장치 추가/제거 hard
> - 기밀 보호 hard
> - 전송 지연 / 중계기 수 많음
> - 데이터 : 단방향 or 양방향 전송

<br/>

### 4) 버스형(Bus)

> **개념** : 한 개 통신 회선, 여러 대의 단말장치 연결
>
> - 단말장치 추가/제거 easy
> - 하나 고장나도 전체에 영향 X => 신뢰성 O
> - 기밀 보호 hard
> - 통신 회선 길이 제한

<br/>

### 5) 계층형(Tree, 분산형)

> **개념** : 중앙 컴퓨터와 일정 지역 단말장치까지는 하나의 통신 회선 / 중간 단말장치로부터 일정 지역 내 이웃하는 단말장치 연결
>
> - 분산 처리 시스템 구성 방식

<br/>

### 6) 망형(Mesh)

> **개념** : 모든 지점의 컴퓨터/단말장치 서로 연결 (노드 연결성 높음)
>
> - 많은 단말장치로부터 많은 양의 통신 필요로 할 때 유리
> - 통신 회선 총 경로 가장 long
> - 노드 수 n
>    - 회선 수 = n(n-1) / 2 
>    - 노드 당 포트 수 = (n-1)

<br/>

### 7) 네트워크 분류

> **근거리 통신망(LAN; Local Area Network)**
> - 가까운 거리의 컴퓨터/프린터/저장장치 등 자원 연결해 구성
> - 주로, 자원 공유 목적
> - 전송 속도 빠르고, 에러 발생율 낮음
> - 주로, 링형 / 버스형
>
> **광대역 통신망(WAN; Wide Area Network)**
> - (국가 간, 대륙 간) 멀리 떨어진 사이트 연결해 구성
> - 통신 속도 느리고, 에러 발생율 높음
> - 일정 지역 근거리 연결 & 각 근거리 연결하는 방식

<br/>

### 8) NAT(Network Address Translation, 네트워크 주소 변환)

> **개념** : 한 개의 정식 IP 주소에 대량의 가상 사설 IP 주소 할당/연결
>
> - 문제 : 외부에 접속할 수 있는 노드가 어느 시점에 한 개로 제한 (한 개의 IP 주소 쓰기 때문)
> - 해결 : IP 마스커레이드 (LINUX 서버를 통해 다른 네트워크에 접속 가능)

<summary>5. 경로 제어 / 트래픽 제어</summary>

<br/>

### 1) 경로 제어(Routing)

> **개념** : 전송 경로 중 최적 패킷 교환 경로를 결정 (by. 경로 제어표, Routing Table)
>
> **경로 제어 요소**
> - 성능 기준
> - 경로 결정 시간/장소
> - 정보 발생지
> - 경로 정보의 갱신 시간

<br/>

### 2) 경로 제어 프로토콜(Routing Protocol)

> **1. IGP(Interior Gateway Protocol, 내부 게이트웨이 프로토콜)** : 하나의 자율 시스템(AS, 도메인) 내 라우팅에 사용
>
> - **RIP(Routing Information Protocol)**
>    - 가장 널리 사용 / 소규모 동종 네트워크(자율 시스템, AS) 내에서 효율적
>    - 최단 경로 탐색(Bellman-Ford 알고리즘) / 거리 벡터 라우팅 프로토콜
>    - 최대 홉(Hop) 수 = 15 (목적지 도달 위해 거치는 네트워크 수)
>    - 30초마다 모든 라우터에 '라우팅 정보' 알림 / 180초 내 새로운 정보 없으면 해당 경로 '이상 상태' 간주
>
> - **OSPF(Open Shortest Path First protocol)**
>    - RIP 단점 해결 / 대규모 네트워크
>    - 노드 간 거리, 링크 상태 정보를 실시간 반영 => 최단 경로 라우팅
>    - 최단 경로 탐색(다익스트라 알고리즘)
>    - 라우팅 변화된 정보만 모든 라우터에 알림
>
> **2. EGP(Exterior Gateway Protocol, 외부 게이트웨이 프로토콜)** : 자율 시스템(AS, 게이트웨이) 간 라우팅
>
> **3. BGP(Border Gateway Protocol)** : 자율 시스템(AS) 간 라우팅 (EGP 보완)
>
> - 초기 BGP 라우터들이 연결될 때는, 전체 경로 제어표(라우팅 테이블) 교환
> - 이후에는, 변화된 정보만 교환

<br/>

### 3) 트래픽 제어(Traffic Control)

> **개념** : 전송되는 패킷 흐름/양 조절
>
> **목적** : 네트워크 보호, 성능 유지, 네트워크 자원의 효율적 이용
>
> **종류**
> - 흐름 제어
> - 폭주(혼잡) 제어
> - 교착상태 방지

<br/>

### 4) 흐름 제어(Flow Control)

> **개념** : 송/수신 간 전송되는 패킷 양/속도 규제
>
> **기법**
> - **정지-대기(Stop-and-Wait)**
>    - 수신 측 확인 신호(ACK) 받은 후, 다음 패킷 전송
>    - 한 번에 하나의 패킷만 전송
>
> - **슬라이딩 윈도우(Sliding Window)**
>    - 확인 신호(수신 통지) 이용해 송신 데이터 양 조절
>    - 확인 신호 없어도, 미리 정해진 패킷 수 만큼 연속 전송 / 한 번에 여러 개 패킷 전송 (효율 good)
>    - 최대 패킷 수 미리 약속 (= Window Size : 상황 따라 변함)

<br/>

### 5) 폭주 제어(Congestion Control)

> **개념** : 네트워크 내 패킷 수 조절 => 네트워크 오버플로(Overflow) 방지
>
> **기법**
> - **느린 시작(Slow Start)** : 윈도우 크기 2배씩 지수 증가 => 갈수록 빨라짐
>    - 전송 데이터 크기가 임계 도달 시, 혼잡 회피 단계
>
> - **혼잡 회피(Congestion Avoidance)** : 윈도우 크기를 1씩 선형 증가
